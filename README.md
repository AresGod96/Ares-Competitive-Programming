## Highlights

For most problems, I will rate their difficulty from **Bronze V** <img src="./boj-icon/bronze5.svg" alt="Bronze 5" width="16" height="16"> (easiest) to **Ruby I** <img src="./boj-icon/ruby1.svg" alt="Ruby 1" width="16" height="16"> (hardest) based on [solved.ac](https://solved.ac) rating system.

Additionally, I will do my best to include a brief editorial for each problem and categorize them with relevant tags.

## Editorials :books:

Many solutions ship with a self-contained **editorial** that lives as a `README.md` next to the solution file. Each editorial follows a consistent template:

- **Header** — judge, problem ID, title, and a [solved.ac](https://solved.ac) difficulty badge.
- **Links** — direct links to the original problem statement and source judge.
- **Tags** — the key techniques, linking to the relevant [`library/`](./library) module when one exists.
- **Solution** — a write-up of *why* the approach works (not just what the code does), with LaTeX math, worked examples, implementation gotchas, and a final **complexity** analysis.

Selected problems also include a **Vietnamese translation** as `README-vi.md` alongside the English `README.md`.

> :pencil: Editorials are generated with the repo's `/write-tutorial` helper, which reads the solution, fills the template, and estimates the difficulty tier against the solved.ac scale.
